# Lint 检查清单

lint 阶段的目标：让 wiki **可持续地保持健康**，而不是一次性"修正"。每次执行后产出一份 `wiki/_meta/lint-<YYYY-MM-DD>.md` 报告，按用户确认逐项修复。

---

## A. 结构性检查

| 检查项 | 怎么做 | 输出 |
|---|---|---|
| **孤儿页**：无任何入站 `[[link]]` | grep 所有 wiki 页面，统计每个 slug 被引用次数 | 列出 0 引用的页面，建议合并或在 index 中分类 |
| **死链**：`[[xxx]]` 指向不存在的页面 | 枚举所有 wikilink，比对实际文件 | 列出，建议建页或修正链接 |
| **空页面 / stub**：内容 < 5 行 | 文件行数检查 | 列出，建议扩展或合并 |
| **目录违规**：页面 type 与所在目录不匹配 | frontmatter.type 对照路径 | 列出，建议移动 |
| **frontmatter 缺失**：必填字段为空 | 解析每个 .md | 列出 |

## B. 内容一致性检查

| 检查项 | 怎么做 | 输出 |
|---|---|---|
| **跨页面矛盾**：同一断言不同结论 | 抽取每页中的"事实陈述"，按主题聚合后比较 | 列出冲突对，附两边引用 |
| **过时声明**：新 source 已覆盖旧说法但旧页未刷新 | 对比 source 时间与页面 updated_at | 列出，建议刷新或显式保留历史观点 |
| **无来源断言**：陈述句缺少 `[[source]]` 或 `(raw/...)` | 正则扫描句子末尾 | 列出，建议补来源或降级为"推断" |
| **冲突未标注**：发现矛盾但页面没有 `> ⚠️` 提示 | 配合 A 项使用 | 列出 |

## C. 覆盖度检查

| 检查项 | 怎么做 | 输出 |
|---|---|---|
| **应建未建**：在 ≥3 个页面中被提及但没有自己的页面的实体/概念 | 词频/实体抽取 | 列出 top-N，建议建页 |
| **数据缺口**：某主题断言稀疏（<5 条） | 按 tags 聚合断言数 | 列出，建议追加新资料 |
| **缺失交叉引用**：A 页提到 B（用其 title/alias），但没有 `[[B]]` | 名称/alias 匹配 | 列出，建议补链 |

## D. 元健康

| 检查项 | 怎么做 | 输出 |
|---|---|---|
| **index.md 与实际文件偏差** | 比对 index 条目与文件系统 | 列出新增未登记/已删除未清理 |
| **log.md 长期未更新** | 检查 log 最后日期 | 提示用户工作流可能脱节 |
| **SCHEMA 偏离**：实际页面与 SCHEMA 约定不一致 | 抽样比对 | 建议要么修页面要么修 SCHEMA |

## Lint 报告输出格式

`wiki/_meta/lint-2026-05-18.md`：

```markdown
---
title: "Lint 报告 2026-05-18"
type: meta
slug: lint-2026-05-18
updated_at: 2026-05-18
---
```

---

## 频率建议

- 小型 wiki（< 50 页）：用户主动触发即可。
- 中型（50–200 页）：建议每 10 次 ingest 跑一次。
- 大型（> 200 页）：建议每周一次，并在 lint 报告页中追踪长期趋势（用 Dataview 或简单脚本）。
